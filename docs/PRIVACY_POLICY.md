# Privacy Policy (Miku)

**Effective date:** 2026-04-05  
**Last updated:** 2026-04-05

> This document is a general template for a Discord bot. It is not legal advice. You should review and customize it for your deployment, jurisdiction, and hosting setup.

## 1) Summary
Miku is a Discord bot that provides leveling and other features. To provide these features, the Bot processes certain Discord-related identifiers and server configuration data.

We do **not** sell personal data.

## 2) Data We Process
Depending on which features you use, we may process and store:

### A) Discord identifiers and server data
- **User ID** (Discord snowflake)
- **Server (Guild) ID**
- **Channel IDs** used for configuration (e.g., a configured level-up announcement channel)
- **Role IDs** used for leveling role rewards

### B) Leveling and usage statistics
- XP totals, level, message count
- Timestamps used to enforce cooldowns / track activity (e.g., last message time)

### C) Message content
- The Bot may **read message events** to award XP and respond to commands.
- The Bot is designed to **not store message content** for leveling.

### D) Logs and diagnostics
- Basic operational logs may include server names/IDs and error information for debugging and abuse prevention.

## 3) How We Use Data
We use the processed data to:
- Provide leveling features (XP, rank cards, leaderboards)
- Store server configuration (e.g., role rewards, announcement channel)
- Respond to commands (utility/info/fun)
- Maintain reliability, prevent abuse, and fix bugs

## 4) Legal Bases (if applicable)
Depending on your jurisdiction, processing may be based on one or more of:
- **Consent** (by inviting/using the Bot)
- **Legitimate interests** (operating and securing the Bot)
- **Contract/performance of service** (providing requested features)

## 5) Sharing and Third Parties
We may share or transmit data to:
- **Discord** (the platform where the Bot operates)
- **Hosting providers** that run the Bot and its database
- **Third-party APIs** used by features (e.g., GitHub API for repository/user lookups)

We do not intentionally share data with advertisers.

## 6) Data Retention
We retain stored data only as long as needed to provide features.

Typical retention rules you can adopt (customize this):
- Leveling data is retained while the Bot is in a server.
- When the Bot is removed from a server, server data may be deleted within a reasonable period.
